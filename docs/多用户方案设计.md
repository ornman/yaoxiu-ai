# 多用户数据存储方案

## 方案一：设备标识隔离（推荐先做）

### 实现成本：⭐⭐（2天）
### 优点：
- 无需登录注册，零门槛使用
- 数据按设备隔离
- 可配合本地加密保护隐私

### 缺点：
- 换设备数据不迁移
- 无法跨设备同步

### 实现步骤：
1. 首次访问生成 device-id（UUID）
2. 所有数据 key 加上 device-id 前缀
3. 提供"导出/导入数据"功能手动迁移

```javascript
// 示例代码
const deviceId = localStorage.getItem('device-id') || generateUUID();
localStorage.setItem('device-id', deviceId);

// 存储时
localforage.setItem(`conv_${deviceId}_${conversationId}`, data);
```

## 方案三：完整用户系统（长期方案）

### 实现成本：⭐⭐⭐⭐⭐（2-3周）
### 数据库选型：

#### A. Cloudflare D1（推荐，零成本）
```sql
-- 用户表
CREATE TABLE users (
    id TEXT PRIMARY KEY,  -- UUID
    phone TEXT UNIQUE,    -- 手机号登录
    password_hash TEXT,   -- 密码
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 对话表
CREATE TABLE conversations (
    id TEXT PRIMARY KEY,
    user_id TEXT REFERENCES users(id),
    title TEXT,
    created_at DATETIME,
    updated_at DATETIME
);

-- 消息表
CREATE TABLE messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    conversation_id TEXT REFERENCES conversations(id),
    role TEXT,  -- 'user' | 'assistant'
    content TEXT,
    timestamp DATETIME
);

-- 收藏表
CREATE TABLE favorites (
    id TEXT PRIMARY KEY,
    user_id TEXT REFERENCES users(id),
    question TEXT,
    answer TEXT,
    tags TEXT,  -- JSON 数组
    created_at DATETIME
);
```

#### B. Supabase（PostgreSQL，免费额度够用）
- 内置身份验证系统
- 实时订阅功能
- 行级安全策略

#### C. Firebase（Google，国内不稳定）

## 方案四：免登录但支持数据迁移

### 实现成本：⭐⭐（3天）

**核心思路**：
1. 数据仍存本地
2. 生成"数据码"用于迁移
3. 类似微信聊天记录迁移

```
用户A: 生成迁移码 (临时存储到后端)
用户B: 输入迁移码 → 下载数据 → 存到本地
```

**临时存储方案**：
- Cloudflare KV（免费 10GB）
- 数据存 24 小时后自动删除
- 端到端加密，后端无法读取

## 🔐 数据隐私考虑

无论哪种方案，都应：
1. **端到端加密** - 敏感内容前端加密后存储
2. **数据脱敏** - 后端日志不记录用户对话内容
3. **用户删除权** - 提供"清空所有数据"功能
4. **本地优先** - 即使后端同步，也应保留本地缓存
