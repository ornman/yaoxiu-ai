本项目采用 **Git Flow + Tag** 的版本管理策略。

## 版本回滚方法

### 回滚到 v1.0.0（当前稳定版）

```bash
# 方式1：切换到 tag（只读，不修改）
git checkout v1.0.0

# 方式2：从 tag 创建修复分支
git checkout -b v1.0.0-hotfix v1.0.0

# 方式3：强制回滚 master 到 v1.0.0（危险操作）
git checkout master
git reset --hard v1.0.0
git push -f origin master
```

### 切换到 v2.0.0 开发分支

```bash
git checkout v2.0.0-dev
```

---

## 版本发布流程

### v2.0.0 发布步骤

```bash
# 1. 确保 v2.0.0-dev 分支代码完整
git checkout v2.0.0-dev

# 2. 合并到 master
git checkout master
git merge v2.0.0-dev --no-ff -m "Release v2.0.0: 心理学改版"

# 3. 打 tag
git tag -a v2.0.0 -m "Release v2.0.0: 心理学改版
- 纹样盲盒系统
- 绣娘等级体系
- 情绪动画系统
- 分享卡片2.0"

# 4. 推送
git push origin master
git push origin v2.0.0
```

## 分支说明

| 分支/Tag | 用途 | 保护级别 |
|----------|------|----------|
| master | 稳定版本，可部署 | 🔒 保护 |
| v1.0.0 | 历史稳定版本 | 🔒 只读 |
| v2.0.0-dev | 开发分支 | ✅ 可修改 |
| v2.0.0 | 发布后将创建 | 🔒 只读 |

---

## 日常开发流程

```bash
# 开始 v2.0.0 开发
git checkout v2.0.0-dev

# 开发新功能...
git add .
git commit -m "feat: 添加盲盒系统"

# 推送开发分支
git push origin v2.0.0-dev

# 如果需要紧急回滚到 v1.0.0
git checkout v1.0.0
```

---

## 重要提示

1. **master 分支永远可部署** - 不要直接在 master 上开发
2. **tag 一旦创建不可修改** - 版本历史永久保留
3. **v2.0.0-dev 是开发分支** - 所有新功能在此开发
4. **随时可回滚** - 通过 tag 可随时回到任意版本
