## 一、系统架构总览

```
用户浏览器 (index.html)
  │
  ├─ [GET]  健康检查 → 获取签名令牌
  ├─ [POST] 聊天请求 → 携带签名令牌
  └─ [POST] 图片识别 → 携带签名令牌
      │
      ▼
Cloudflare Worker (worker.js)
  ├─ CORS 白名单校验
  ├─ IP 封禁检查（KV）
  ├─ 速率限制（KV 分布式 / 内存降级）
  ├─ 请求签名验证
  ├─ 异常请求检测
  ├─ 审计日志写入（KV）
  └─ 输入验证 + 注入检测
      │
      ▼
AI 服务商 (DeepSeek / Qwen)
```

### 安全特性清单

| 特性 | 位置 | KV 依赖 | 可关闭 |
|------|------|---------|--------|
| CORS 域名白名单 | Worker | 否 | 否 |
| 速率限制 | Worker | 推荐 | 否 |
| 输入验证 + 注入检测 | Worker | 否 | 否 |
| 请求签名（令牌模式） | Worker + 前端 | 否 | 是 |
| 异常监控 + 自动封禁 | Worker | 是 | 部分 |
| 审计日志 | Worker | 是 | 部分 |
| XSS 防护 (DOMPurify) | 前端 | 否 | 否 |
| CSP 策略 | 前端 | 否 | 否 |
| CDN SRI 完整性 | 前端 | 否 | 否 |
| 安全响应头 | Worker | 否 | 否 |

## 二、首次部署配置

### 2.1 创建 Cloudflare KV 命名空间

KV 用于分布式速率限制、审计日志和自动封禁。**不创建也能运行**，但降级为单节点内存模式。

```bash
# 创建 KV 命名空间
npx wrangler kv namespace create SECURITY_KV
# 输出类似：id = "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

创建后，编辑 `wrangler.toml` 添加绑定：

```toml
[[kv_namespaces]]
binding = "SECURITY_KV"
id = "上面输出的 id"
```

重新部署：

```bash
npx wrangler deploy
```

### 2.2 配置密钥和令牌

```bash
# 必需：AI 服务商密钥
echo "sk-你的DeepSeek密钥" | npx wrangler secret put DEEPSEEK_API_KEY
echo "sk-你的Qwen密钥"     | npx wrangler secret put QWEN_API_KEY

# 可选：请求签名密钥（设置后启用签名验证）
echo "$(openssl rand -hex 32)" | npx wrangler secret put REQUEST_SIGNING_SECRET
```

### 2.3 验证部署

```bash
# 健康检查
curl -s https://api.yaoxiumax.top/ | python3 -m json.tool

# 期望输出包含：
# "version": "3.0-secure"
# "features": ["rate-limit", "input-validation", "cors-restriction",
#              "request-signing", "anomaly-monitor", "audit-log"]

# 安全头检查
curl -sD - https://api.yaoxiumax.top/ -o /dev/null | grep -iE \
  "x-frame|strict-transport|x-content-type|referrer-policy|permissions-policy"
```

## 三、日常运维操作

### 3.1 查看审计日志

审计日志存储在 KV 中，key 格式为 `audit:ISO时间戳-随机ID`。

```bash
# 列出最近的审计日志 key
npx wrangler kv key list --namespace-id="你的KV_ID" --prefix="audit:" | \
  python3 -c "import sys,json; keys=json.load(sys.stdin); [print(k['name']) for k in keys[-20:]]"

# 读取某条审计日志
npx wrangler kv get --namespace-id="你的KV_ID" "audit:2026-04-25T10-30-00-abc123"
```

#### 审计日志字段说明

```json
{
  "id": "audit:2026-04-25T10-30-00-abc123",
  "timestamp": "2026-04-25T10:30:00.123Z",
  "event": "REQUEST | BLOCKED | AUTO_BAN",
  "ip": "1.2.3.4",
  "method": "POST",
  "type": "chat-context",
  "model": "deepseek-chat",
  "blocked": "rate-limit | signature-failed | anomaly",
  "alerts": []
}
```

#### event 类型

| event | 含义 |
|-------|------|
| `REQUEST` | 正常请求 |
| `BLOCKED` | 被安全策略阻止的请求 |
| `AUTO_BAN` | IP 被自动封禁事件 |

### 3.2 手动封禁/解封 IP

```bash
# 封禁 IP（1小时）
# 在 Cloudflare Dashboard → Workers → KV 查看器中手动添加
# Key:   ban:1.2.3.4
# Value: {"reason":"手动封禁","expiresAt":1745500000000,"alertCount":99}
# TTL:   3600

# 解封 IP
npx wrangler kv delete --namespace-id="你的KV_ID" "ban:1.2.3.4"
```

### 3.3 查看速率限制状态

```bash
# 查看某 IP 的速率限制计数
npx wrangler kv get --namespace-id="你的KV_ID" "rate:1.2.3.4"

# 重置某 IP 的速率限制
npx wrangler kv delete --namespace-id="你的KV_ID" "rate:1.2.3.4"
```

### 3.4 查看 nonce 缓存（防重放）

```bash
# Nonce 缓存自动过期（TTL 600秒），一般不需要手动操作
# 如果需要清理所有 nonce：
npx wrangler kv key list --namespace-id="你的KV_ID" --prefix="nonce:" | \
  python3 -c "import sys,json; [print(k['name']) for k in json.load(sys.stdin)]" | \
  while read key; do
    npx wrangler kv delete --namespace-id="你的KV_ID" "$key" --force
  done
```

## 四、配置调整

### 4.1 修改速率限制参数

编辑 `worker.js` 中的 `SECURITY_CONFIG`：

```javascript
RATE_LIMIT: {
  MAX_REQUESTS: 30,              // 每分钟最大请求数（默认 30）
  WINDOW_MS: 60 * 1000,          // 时间窗口（默认 60秒）
  BLOCK_DURATION_MS: 5 * 60 * 1000, // 封禁时长（默认 5分钟）
},
```

修改后重新部署：

```bash
npx wrangler deploy
```

### 4.2 修改自动封禁阈值

```javascript
MONITOR: {
  AUTO_BAN_THRESHOLD: 100,       // 每分钟请求数触发阈值
  AUTO_BAN_DURATION_MS: 60 * 60 * 1000, // 封禁时长（默认 1小时）
},
```

### 4.3 开关请求签名

```javascript
SIGNING: {
  ENABLED: true,  // 改为 false 可关闭签名验证
},
```

或者不设置 `REQUEST_SIGNING_SECRET`（未配置时自动跳过）。

### 4.4 添加/移除 CORS 域名

```javascript
ALLOWED_ORIGINS: [
  'https://yaoxiumax.top',
  'https://www.yaoxiumax.top',
  'https://yaoxiu-ai.vercel.app',
  // 添加新域名：
  // 'https://new-domain.com',
],
```

### 4.5 调整审计日志保留期

```javascript
AUDIT: {
  KV_TTL: 7 * 86400,  // 默认 7 天，可改为 30 * 86400（30天）
},
```

## 五、监控与告警

### 5.1 关键指标

通过 Cloudflare Dashboard → Workers → Metrics 查看：

| 指标 | 正常范围 | 异常阈值 | 动作 |
|------|----------|----------|------|
| 请求数/分钟 | < 100 | > 1000 | 检查是否有攻击 |
| 429 响应比例 | < 1% | > 10% | 速率限制可能过严 |
| 403 响应比例 | < 0.1% | > 5% | 可能被扫描攻击 |
| 401 响应比例 | < 0.1% | > 1% | 签名配置可能有问题 |
| Worker 错误率 | < 0.1% | > 1% | 检查日志排查 |

### 5.2 Cloudflare Logpush 配置（可选）

如果需要将审计日志推送到外部系统：

1. Cloudflare Dashboard → Workers → Logs → Logpush
2. 配置目标（S3、Splunk、Datadog 等）
3. Worker 的 `console.log()` 输出会自动推送

### 5.3 GitHub Actions 安全 CI

每次推送到 `master` 或 `redesign-ui` 分支时自动运行：

| 检查项 | 作用 | 失败时 |
|--------|------|--------|
| gitleaks | 检测密钥泄露 | 阻止合并 |
| 安全头检查 | 验证生产 API 响应头 | 告警 |
| CSP 验证 | 检查前端 CSP 配置 | 告警 |
| SRI 审计 | 检查 CDN 脚本完整性 | 告警 |

查看 CI 结果：

```bash
gh run list --workflow=security.yml --limit=5
```

## 六、应急响应

### 6.1 发现密钥泄露

```bash
# 1. 立即轮换密钥
echo "新的密钥" | npx wrangler secret put DEEPSEEK_API_KEY

# 2. 清理 Git 历史（如果密钥已提交）
# 使用 BFG Repo-Cleaner：
git clone --mirror https://github.com/ornman/yaoxiu-ai.git
java -jar bfg.jar --replace-text replacements.txt yaoxiu-ai.git
cd yaoxiu-ai.git
git reflog expire --expire=now --all
git gc --prune=now --aggressive
git push --force
```

### 6.2 遭遇 DDoS 攻击

```bash
# 1. 临时降低速率限制（编辑 worker.js）
MAX_REQUESTS: 10,  # 从 30 降到 10
BLOCK_DURATION_MS: 30 * 60 * 1000,  # 封禁延长到 30 分钟

# 2. 紧急部署
npx wrangler deploy

# 3. 在 Cloudflare Dashboard → Security → WAF 中：
#    - 开启 Under Attack 模式
#    - 添加 Rate Limiting 规则
#    - 启用 Bot Management
```

### 6.3 AI 服务被滥用

```bash
# 1. 查看最近的审计日志，找到异常 IP
npx wrangler kv key list --namespace-id="你的KV_ID" --prefix="audit:" | tail -100

# 2. 手动封禁异常 IP
# 通过 Cloudflare Dashboard KV 管理器添加 ban:IP_KEY

# 3. 临时关闭服务
# 在 Cloudflare Dashboard → Workers → Routes 中禁用路由
```

### 6.4 签名验证导致正常用户无法使用

```bash
# 方案 A：关闭签名验证
# 编辑 worker.js: SIGNING.ENABLED = false
# 然后 npx wrangler deploy

# 方案 B：删除签名密钥（签名自动跳过）
npx wrangler secret delete REQUEST_SIGNING_SECRET
```

### 6.5 回滚到上一版本

```bash
# 查看 Worker 部署历史
npx wrangler deployments list

# 回滚到指定版本
npx wrangler rollback <version-id>

# 或使用 Git 回滚代码
git log --oneline -5  # 找到之前的 commit
git revert HEAD       # 回滚最后一次提交
npx wrangler deploy   # 重新部署
```

## 八、KV 数据管理

### 8.1 存储用量

| Key 前缀 | 用途 | TTL | 估计数量 |
|----------|------|-----|----------|
| `rate:` | 速率限制计数 | 120s | 与活跃 IP 数相当 |
| `ban:` | IP 封禁记录 | 3600s | 极少 |
| `alert:` | 告警计数 | 120s | 与异常 IP 数相当 |
| `nonce:` | 防重放缓存 | 600s | 与请求数相当 |
| `audit:` | 审计日志 | 604800s (7天) | 取决于流量 |

### 8.2 清理过期数据

KV TTL 过期后 Cloudflare 自动清理，无需手动操作。

如需手动清理审计日志：

```bash
# 删除 7 天前的审计日志（TTL 已自动处理，通常不需要）
npx wrangler kv key list --namespace-id="你的KV_ID" --prefix="audit:" | \
  python3 -c "
import sys, json
from datetime import datetime, timedelta
cutoff = (datetime.utcnow() - timedelta(days=7)).isoformat()[:10]
keys = json.load(sys.stdin)
old = [k['name'] for k in keys if k['name'][6:16] < cutoff]
print(f'{len(old)} old entries to clean')
for k in old: print(k)
"
```

## 十、常见问题

### Q: Worker 报 "Service configuration error"
A: AI 服务商密钥未配置。运行：
```bash
echo "你的密钥" | npx wrangler secret put DEEPSEEK_API_KEY
```

### Q: 前端请求返回 401 "请求验证失败"
A: 签名验证失败。检查 `REQUEST_SIGNING_SECRET` 是否已配置但前端获取令牌失败。临时方案：删除该 secret 让系统跳过签名。

### Q: 正常用户被速率限制
A: 调高 `MAX_REQUESTS` 或清除该 IP 的 KV 记录：
```bash
npx wrangler kv delete --namespace-id="你的KV_ID" "rate:用户IP"
```

### Q: KV 未绑定会有什么影响？
A: 以下功能降级：
- 速率限制：从跨节点共享退化为单节点内存模式
- 审计日志：不写入（仅在控制台输出）
- 异常监控：不记录告警和封禁
- Nonce 防重放：跳过

核心功能（CORS、输入验证、注入检测、消息角色过滤、安全头）不受影响。

### Q: 如何临时关闭某个安全特性？
A: 编辑 `worker.js` 中的 `SECURITY_CONFIG`：

| 特性 | 开关 |
|------|------|
| 请求签名 | `SIGNING.ENABLED: false` |
| 注入检测 | 删除 `INJECTION_PATTERNS` 中的模式 |
| 速率限制 | 无法完全关闭，可设 `MAX_REQUESTS: 9999` |

修改后 `npx wrangler deploy`。

### Q: 如何添加新的注入检测规则？
A: 在 `worker.js` 的 `INJECTION_PATTERNS` 数组中添加正则表达式：
```javascript
/新的注入模式/i,
```
