# 国内无墙部署指南 - 绑定自定义域名

## 问题现象
- ✅ 网站能正常打开（https://yaoxiumax.top）
- ❌ 不开梯子无法对话（Worker API 被墙）
- 错误提示：连接超时、网络错误、API 不可用

## 解决方案概述

**核心问题**：`*.workers.dev` 域名在国内某些网络环境下会被拦截

**解决方案**：给 Cloudflare Worker 绑定自定义域名 `api.yaoxiumax.top`
- ✅ 国内直连无墙
- ✅ 访问速度更快
- ✅ 更专业的接口地址

---

## 前置条件检查

### 1. 确认域名托管状态
打开 https://dash.cloudflare.com，看左侧域名列表：

| 状态 | 操作 |
|------|------|
| 已有 `yaoxiumax.top` | ✅ 直接跳到步骤二 |
| 没有该域名 | ⚠️ 需要先添加域名（见下方） |

### 2. 如果域名不在 Cloudflare（首次添加）

#### Step 2.1: 添加站点
1. 点击 **"Add a Site"**（蓝色按钮）
2. 输入域名：`yaoxiumax.top`
3. 选择计划：**Free**（免费完全够用）
4. 点击 **Continue**

#### Step 2.2: 修改域名 DNS
Cloudflare 会显示两个 Nameserver 地址，类似：
```
chad.ns.cloudflare.com
michelle.ns.cloudflare.com
```

**去域名注册商修改**（你在哪里买的域名）：
- 阿里云域名：登录阿里云 → 域名控制台 → 找到 `yaoxiumax.top` → DNS 修改
- 腾讯云域名：登录腾讯云 → 域名注册 → 管理 → DNS 服务器
- 其他注册商：找到 DNS 管理或 Nameserver 设置

**把原来的 DNS 改成 Cloudflare 提供的两个地址**

#### Step 2.3: 等待生效
- 通常 **5-30 分钟**生效
- 最长可能需要 **24 小时**
- Cloudflare 会邮件通知你

## 步骤一：在 Cloudflare 添加 Worker 自定义域名

### 1.1 进入 Worker 管理页面
1. 打开 https://dash.cloudflare.com
2. 点击左侧菜单 **"Workers & Pages"**
3. 找到并点击你的 Worker：`yaoembroidery-api`

### 1.2 添加自定义域名
1. 点击顶部标签 **"Triggers"**（触发器）
2. 找到 **"Custom Domains"** 区域
3. 点击 **"Add Custom Domain"**（添加自定义域名）
4. 输入：`api.yaoxiumax.top`
5. 点击 **"Add Custom Domain"**

### 1.3 等待生效
- 状态会显示 **"Verifying..."**（验证中）
- 等待变成 **"Active"**（绿色 ✅）
- 通常 **1-5 分钟**完成

### 1.4 测试验证
浏览器访问：
```
https://api.yaoxiumax.top
```

**成功响应**：
```json
{
  "status": "ok",
  "message": "小瑶服务运行正常 🧵",
  "timestamp": "2026-04-05T...",
  "version": "2.0"
}
```

**如果显示上面内容** → ✅ Worker 配置完成！

## 常见问题排查

### ❌ 问题1：Custom Domain 一直显示 "Verifying"

**可能原因**：
- 域名 DNS 还没切到 Cloudflare
- DNS 缓存未刷新

**解决方法**：
```powershell
# Windows 命令行刷新 DNS 缓存
ipconfig /flushdns
```

或者等待 10-30 分钟再试

---

### ❌ 问题2：api.yaoxiumax.top 打不开

**检查步骤**：
1. **检查 DNS 是否生效**
   ```powershell
   nslookup api.yaoxiumax.top
   ```
   应该返回 Cloudflare 的 IP（104.x.x.x 或 172.x.x.x）

2. **检查 Worker 是否正常**
   访问 `https://yaoembroidery-api.kdy233.workers.dev`
   如果能打开，说明 Worker 没问题

3. **检查域名是否正确添加**
   确认输入的是 `api.yaoxiumax.top` 不是 `yaoxiumax.top`

---

### ❌ 问题3：网站能开，但发消息报错

**可能原因**：
- 前端代码没改对
- 浏览器缓存了旧代码

**解决方法**：
1. **强制刷新页面**
   - Windows：`Ctrl + Shift + R`
   - Mac：`Cmd + Shift + R`

2. **清除浏览器缓存**
   - Chrome：设置 → 隐私和安全 → 清除浏览数据 → 选"缓存"

3. **检查代码是否修改正确**
   打开 `index.html` 第 1212 行，确认是：
   ```javascript
   const DEFAULT_API_URL = 'https://api.yaoxiumax.top';
   ```

---

### ❌ 问题4：部署到 Vercel 时提示 "token is not valid"

**解决方法**：
```powershell
# 重新登录
vercel logout
vercel login

# 然后重新部署
vercel --prod
```
