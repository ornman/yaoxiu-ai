# 本地开发指南

## 项目架构

```
瑶绣项目 = 前端(index.html) + 后端(worker.js)
```

| 部分 | 文件 | 部署位置 | 说明 |
|------|------|----------|------|
| 前端 | `index.html` + `assets/` | Vercel | 纯静态网站 |
| 后端 | `worker.js` | Cloudflare | AI接口+图片识别 |

**两者独立：** 修改前端不需要改后端，反之亦然。

---

## 本地预览 v2.0.0

### 方法1：直接打开（最简单）

```bash
# 1. 确保在 v2.0.0-dev 分支
git checkout v2.0.0-dev

# 2. 直接用浏览器打开 index.html
# 双击 index.html 文件即可
```

**缺点：**
- 部分浏览器限制本地文件访问（如图片上传可能有问题）
- URL 显示为 file:// 而不是 http://

---

### 方法2：本地服务器（推荐）

#### 方式A：Python（如果有Python）

```bash
# 在项目根目录执行
python -m http.server 8080

# 然后访问 http://localhost:8080
```

#### 方式B：Node.js（如果有Node）

```bash
# 安装简单服务器
npm install -g serve

# 在项目根目录执行
serve -p 8080

# 然后访问 http://localhost:8080
```

#### 方式C：VS Code Live Server（最推荐）

1. 安装 VS Code 插件 "Live Server"
2. 右键点击 `index.html` → "Open with Live Server"
3. 自动打开浏览器，地址是 `http://127.0.0.1:5500`

**优点：**
- 自动刷新（保存文件后浏览器自动刷新）
- 支持所有功能（图片上传等）
- URL 是 http://，和线上环境一致

## 管理多个 JS 文件

### 当前文件结构

```
assets/
├── v2-system.js      # v2核心：盲盒+等级+签到系统
├── v2-ui.js          # v2 UI：弹窗+组件
├── v2-styles.css     # v2样式
└── context-manager.js # v1的上下文管理（已有）
```

### 引入顺序（在 index.html 中）

```html
<!-- 先引入样式 -->
<link rel="stylesheet" href="assets/v2-styles.css">

<!-- 原有的 -->
<script src="assets/context-manager.js"></script>

<!-- v2 核心系统（先） -->
<script src="assets/v2-system.js"></script>

<!-- v2 UI组件（后，依赖核心系统） -->
<script src="assets/v2-ui.js"></script>
```

**顺序很重要：** 先有数据（system），后有界面（ui）

## 关于 JS 文件管理

### 方案A：保持分离（当前做法）

**优点：**
- 代码清晰，好维护
- 按需加载（虽然这里是全部加载）

**缺点：**
- 多一个 HTTP 请求（实际上影响不大）

### 方案B：合并为一个文件

如果希望只有一个 JS 文件：

```bash
# 手动合并（简单做法）
cat assets/v2-system.js assets/v2-ui.js > assets/v2-bundle.js
```

然后在 index.html 中只引入：
```html
<script src="assets/v2-bundle.js"></script>
```

**建议：** 保持分离，除非性能成为问题。

## 如何启用 v2.0.0（详细步骤）

### 步骤1：切换到 v2.0.0-dev 分支

```bash
git checkout v2.0.0-dev
```

### 步骤2：修改 index.html

在 `index.html` 的 `<head>` 部分（style 标签附近）添加：

```html
<!-- v2.0.0 样式 -->
<link rel="stylesheet" href="assets/v2-styles.css">
```

在 `index.html` 的 `</body>` 前，找到其他 script 标签，添加：

```html
<!-- v2.0.0 核心系统 -->
<script src="assets/v2-system.js"></script>
<script src="assets/v2-ui.js"></script>
```

### 步骤3：添加初始化代码

在现有的 `DOMContentLoaded` 事件中，添加 v2 初始化：

```javascript
document.addEventListener('DOMContentLoaded', () => {
    // ... 原有的初始化代码 ...
    
    // ===== v2.0.0 初始化 =====
    // 初始化等级徽章
    LevelBadge.init();
    
    // 初始化签到组件
    CheckinComponent.init();
    
    // 添加盲盒入口按钮
    const headerActions = document.querySelector('header .flex.items-center.gap-1');
    if (headerActions) {
        headerActions.insertAdjacentHTML('beforeend', `
            <button onclick="BlindBoxModal.open()" class="blindbox-entry-btn" title="得纹样">
                <span>🎁</span>
                <span class="blindbox-count">${V2System.blindBox.getDailyDraws()}</span>
            </button>
        `);
    }
    
    // 修改收藏按钮行为（改为打开收藏册）
    const favBtn = document.getElementById('headerFavBtn');
    if (favBtn) {
        favBtn.onclick = () => PatternCollection.open();
    }
});
```

### 步骤4：添加盲盒按钮样式

在 `index.html` 的 `<style>` 部分添加：

## 切换回 v1.0.0

如果不想用 v2.0.0 了：

```bash
git checkout master

# 刷新浏览器即可
```

## 部署到 Vercel 前的检查

### 检查清单

- [ ] 所有 JS 文件都已提交到 git
- [ ] index.html 中正确引入了 v2 文件
- [ ] 本地预览功能正常
- [ ] 盲盒能正常抽取
- [ ] 等级显示正常
- [ ] 移动端显示正常

### 部署步骤

```bash
# 1. 提交所有更改
git add .
git commit -m "feat: enable v2.0.0 features"

# 2. 推送到 GitHub
git push origin v2.0.0-dev

# 3. Vercel 会自动部署（如果已绑定 GitHub）
# 或者手动在 Vercel 控制台选择 v2.0.0-dev 分支预览

# 4. 确认无误后，合并到 master
git checkout master
git merge v2.0.0-dev
git push origin master
```

## 常见问题

### Q: 本地打开时 JS 报错？
**A:** 可能是文件路径问题。确保 `assets/v2-system.js` 等文件存在，且路径正确。

### Q: 样式没有生效？
**A:** 检查 `v2-styles.css` 是否正确引入，且 CSS 变量（如 `--gold`）已定义。

### Q: 想和线上版本对比？
**A:** 
- 线上版本：`git checkout master`
- v2版本：`git checkout v2.0.0-dev`
- 用不同浏览器标签同时打开对比

### Q: 数据会冲突吗？
**A:** v1 和 v2 使用相同的 localStorage key，但 v2 新增了字段，不会破坏 v1 的数据。
