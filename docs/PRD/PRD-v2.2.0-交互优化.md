# 瑶绣AI v2.2.0 交互优化 PRD

## 1. 个人成长系统重构

### 1.1 需求概述
将现有的等级徽章点击直接打开绣谱的交互，改为先显示个人成长下拉菜单，再提供绣谱入口。

### 1.2 详细设计

#### 等级徽章交互
- **点击行为**：点击等级徽章不直接打开绣谱，而是显示下拉菜单
- **下拉菜单内容**：
  - 当前等级图标和名称
  - 等级进度条（当前XP/下一级所需XP）
  - 学习能力统计（已掌握技能、学习天数等）
  - 快捷入口：我的绣谱、纹样盲盒

#### 下拉菜单样式
```
┌─────────────────────────┐
│  [等级图标] 学徒         │
│  ████████████░░░ 75%    │
│  125/200 XP             │
│  ─────────────────────  │
│  📚 已掌握: 5个纹样      │
│  📅 学习天数: 3天        │
│  ─────────────────────  │
│  📖 我的绣谱  →         │
│  🎁 纹样盲盒  →         │
└─────────────────────────┘
```

#### 绣谱入口位置
1. **个人成长下拉菜单**：主要入口
2. **盲盒界面右上角**：添加"我的绣谱"链接

### 1.3 实现要点
- 新建 `openGrowthMenu()` 函数
- 修改 `openLevelModal()` 逻辑

## 2. 抽卡动画修复与优化

### 2.1 问题修复
**当前问题**：卡片旋转后显示反过来的（背面文字倒置）

**原因**：CSS 3D翻转动画中，`rotateY(180deg)` 时背面的 `transform` 属性导致内容倒置

**修复方案**：
```css
.blindbox-pattern {
    /* 背面初始状态 */
    transform: rotateY(180deg);
    backface-visibility: hidden; /* 关键修复 */
}
```

### 2.2 动画增强
增加以下动画效果：

#### 抽卡前 - 盒子摇晃
```css
@keyframes boxShake {
    0%, 100% { transform: translateX(0) rotate(0deg); }
    20% { transform: translateX(-8px) rotate(-3deg); }
    40% { transform: translateX(8px) rotate(3deg); }
    60% { transform: translateX(-5px) rotate(-2deg); }
    80% { transform: translateX(5px) rotate(2deg); }
}
```

#### 抽卡中 - 光效爆发
- 添加金色粒子爆炸效果
- 卡片周围发光闪烁

#### 抽卡后 - 结果展示
- 稀有度越高，光效越强
- 传奇品质添加彩虹边框动画

## 3. 消息操作按钮优化

### 3.1 用户消息操作按钮

#### 参考图片样式
- 无背景框，纯文字+图标
- 鼠标悬停时丝滑显现
- 灰色文字，悬停变主题色

#### 布局调整
- 按钮组位于用户消息气泡**右下方**
- 不超出对话栏宽度
- 按钮间距紧凑（8px）

#### 按钮顺序（用户消息）
```
复制 | 编辑 | 分享 | 删除
```
- **复制**：复制消息文本
- **编辑**：修改用户问题（仅视觉修改，需重新生成回答）
- **分享**：生成分享卡片
- **删除**：删除该条消息及AI回复

### 3.2 AI消息操作按钮

#### 按钮顺序（AI消息）
```
复制 | 收藏 | 分享 | 重新生成 | 追问 | 删除
```

#### 样式统一
- 与用户消息按钮样式一致
- 位于AI消息气泡**左下方**

## 4. 暂停/停止对话后操作

### 4.1 需求
当用户点击"停止生成"或对话出错时，已生成的内容也需要支持操作。

### 4.2 实现方案
在以下场景添加操作按钮：
1. 用户点击停止生成
2. 对话发生错误
3. 对话正常完成（已有）

### 4.3 停止生成后的操作按钮
显示：**复制** | **重新生成** | **删除**

- **复制**：复制已生成的内容
- **重新生成**：清除当前AI回复，重新生成（保留用户问题）
- **删除**：删除该条对话

---

## 5. 重新生成逻辑

### 5.1 交互流程
1. 用户点击"重新生成"
2. 保留用户消息不变
3. 删除当前AI回复DOM元素
4. 从chatHistory中移除最后一条assistant消息
5. 调用sendMessage()重新生成

### 5.2 视觉表现
- 用户消息保持原位不动
- AI回复区域显示"正在重新生成..."
- 生成完成后替换内容
