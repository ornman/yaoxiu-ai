# 瑶绣·智问 技术架构说明

## 一、API 调用逻辑

### 1. 两种调用模式

#### 模式 A：前端直连 DeepSeek API（本地测试）

```
┌─────────────┐     HTTP POST      ┌─────────────────┐
│  浏览器     │ ─────────────────→ │  DeepSeek API   │
│  index.html │  Authorization     │  api.deepseek   │
│             │  Bearer sk-xxx     │  .com           │
└─────────────┘                    └─────────────────┘
```

**代码逻辑**：
```javascript
// 前端直接发送请求
fetch('https://api.deepseek.com/chat/completions', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
        'Authorization': 'Bearer sk-your-api-key'  // 密钥暴露！
    },
    body: JSON.stringify({
        model: 'deepseek-chat',  // 或 'deepseek-reasoner'
        messages: [
            { role: 'system', content: '系统提示词' },
            { role: 'user', content: '用户问题' }
        ],
        stream: true  // 流式输出
    })
})
```

**缺点**：API Key 暴露在浏览器里，任何人 F12 都能看到

---

#### 模式 B：通过 Cloudflare Worker 代理（生产环境）

```
┌─────────────┐     HTTP POST      ┌──────────────┐     HTTPS      ┌─────────────────┐
│  浏览器     │ ─────────────────→ │  Cloudflare  │ ────────────→ │  DeepSeek API   │
│  index.html │  无敏感信息        │  Worker      │  密钥安全存储 │  api.deepseek   │
│             │                    │  workers.dev │  在服务端     │  .com           │
└─────────────┘                    └──────────────┘                └─────────────────┘
```

**前端代码**：
```javascript
fetch('https://your-worker.workers.dev', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
        message: '用户问题',
        model: 'deepseek-chat'
    })
})
```

**Worker 代码**（后端）：
```javascript
// 从环境变量读取密钥（安全）
const apiKey = env.DEEPSEEK_API_KEY;

// 转发请求到 DeepSeek
fetch('https://api.deepseek.com/chat/completions', {
    headers: { 'Authorization': `Bearer ${apiKey}` },
    // ...
})
```

**优点**：密钥在服务端，前端不可见

## 二、系统提示词设计

### 完整提示词（worker.js 中）

```markdown
你是瑶族刺绣知识专家，严格遵守以下规则：

【核心专长】
1. 八角花纹样：族徽图腾的几何美学、对称构图、象征意义（太阳纹、蛙纹衍变）
2. 十字挑花针法：反面挑花正面看的绝技、数纱而绣、不打底稿的传统技法
3. 黑底红彩配色：经典色彩体系（黑/红/黄/白/绿）、色彩禁忌与寓意

【回答风格】
- 语言专业且通俗易懂
- 适当提及瑶族支系差异（盘瑶、布努瑶、茶山瑶等）
- 可补充刺绣在服饰中的应用部位（头巾、腰带、袖口、绑腿）

【严格限制】
- 只回答与瑶族刺绣、瑶族文化、民族工艺相关的问题
- 对于无关问题，礼貌拒绝："我是瑶族刺绣专家，只回答刺绣相关问题。如需了解其他话题，请咨询其他专家。"
- 绝不回答编程、政治、娱乐、财经等无关内容
```

### 提示词工程思路

| 元素 | 作用 | 示例 |
|------|------|------|
| **角色定义** | 设定专家人设 | "瑶族刺绣知识专家" |
| **核心专长** | 限定知识范围 | 八角花、十字挑花、黑底红彩 |
| **回答风格** | 控制输出语气 | 专业且通俗易懂 |
| **严格限制** | 防止越狱/滥用 | 只回答刺绣相关问题 |

### 快速/深度模式切换

**快速模式**（V3）：
- 模型：`deepseek-chat`
- 特点：响应快、成本低、适合简单问答

**深度模式**（R1）：
- 模型：`deepseek-reasoner`
- 特点：带思考过程、逻辑严密、适合复杂问题
- 特殊字段：`reasoning_content`（思考链）

## 三、流式输出处理

### SSE（Server-Sent Events）格式

```
data: {"choices":[{"delta":{"content":"八角"}}]}
data: {"choices":[{"delta":{"content":"花纹样"}}]}
data: {"choices":[{"delta":{"content":"是"}}]}
data: [DONE]
```

### 前端处理逻辑

```javascript
const reader = response.body.getReader();
while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    
    // 解析每一行数据
    const lines = decoder.decode(value).split('\n');
    for (const line of lines) {
        if (line.startsWith('data:')) {
            const data = line.slice(5);
            if (data === '[DONE]') continue;
            
            const json = JSON.parse(data);
            const delta = json.choices[0].delta;
            
            // 处理思考过程（R1模型）
            if (delta.reasoning_content) {
                // 显示在"思考中..."区域
            }
            
            // 处理正式回答
            if (delta.content) {
                // 追加到聊天界面
            }
        }
    }
}
```

## 四、安全设计

### 为什么一定要用 Worker？

| 风险 | 前端直连 | Worker代理 |
|------|----------|------------|
| API Key 泄露 | ❌ 暴露 | ✅ 隐藏 |
| 额度被盗用 | ❌ 高风险 | ✅ 可控 |
| 提示词被篡改 | ❌ 可修改 | ✅ 服务端固定 |
| 响应被劫持 | ❌ 无保护 | ✅ HTTPS |

### Worker 的安全措施

1. **密钥存储**：使用 Cloudflare 环境变量
2. **请求验证**：可添加 CORS 限制、速率限制
3. **内容过滤**：可在服务端二次校验
4. **日志监控**：记录异常请求

## 五、本地 vs 生产

| 场景 | 模式 | 配置 |
|------|------|------|
| 本地开发 | 前端直连 | `USE_DIRECT_API = true` |
| 生产部署 | Worker代理 | `USE_DIRECT_API = false` + 部署 Worker |

**切换方式**：修改 `index.html` 第 30 行

```javascript
const USE_DIRECT_API = false;  // 上线前改为 false
```
